# 产品需求文档 (PRD)：校园/社区失物招领平台

## 1. 项目愿景
打造一个信息对称、响应迅速的失物招领平台。通过直观的双栏设计，降低用户获取信息的成本，帮助失主快速找回财物。

## 2. 页面布局与交互 (参考 image_58c67e.png)

### 2.1 首页 (Home Page)
*   **顶部导航栏：**
    *   **左侧：** 平台 Logo 及名称。
    *   **中间：** 搜索框（支持关键词检索物品名称、地点）。
    *   **右侧：** “我要挂失”按钮、“我发现了失物”按钮、“退出登录”按钮、用户头像（点击进入个人中心）。
*   **主体内容区：**
    *   **左侧：失物招领栏 (Found Items)** —— 展示他人捡到并上传的物品。
        *   卡片包含：物品图片、标题、时间、地点。
        *   交互：点击**“我要认领”**，弹出模态框展示拾得者的联系方式（手机号/微信）。
    *   **右侧：失物寻物栏 (Lost Items)** —— 展示失主丢失并寻找的物品。
        *   卡片包含：物品图片（或占位图）、标题、时间、地点。
        *   交互：点击**“联系失主”**，弹出模态框展示失主的联系方式。

### 2.2 发布页面 (Posting Page)
*   **表单字段：** 
    *   信息类型（招领/挂失）。
    *   物品分类（如：证件、电子产品、生活用品、其他）。
    *   物品名称、描述详情。
    *   图片上传（必填/选填）。
    *   丢失/拾获地点。
    *   **联系方式（默认读取个人资料，支持单次修改）。**

### 2.3 用户中心 (User Center)
*   **个人资料：** 修改头像（支持图片上传）、修改密码、修改常用联系方式。
*   **我的发布：** 列表展示自己发布的帖子，支持编辑、删除，以及一键切换状态为“已解决”。

### 2.4 管理后台 (Admin Portal)
*   **内容管理：** 管理员可查看全站帖子，支持强制删除违规信息（如虚假广告、不当内容）。
*   **用户管理：** 查看注册用户列表，支持封禁违规账号。

## 3. 数据库设计 (MySQL)

### 3.1 用户表 (`users`)
| 字段名 | 类型 | 描述 |
| :--- | :--- | :--- |
| `id` | INT (PK) | 用户唯一标识 |
| `username` | VARCHAR | 登录账号 |
| `password` | VARCHAR | 加密后的密码 |
| `avatar` | VARCHAR | 头像 URL |
| `contact` | VARCHAR | 默认联系方式 |
| `role` | ENUM | 角色 (user/admin) |

### 3.2 帖子表 (`items`)
| 字段名 | 类型 | 描述 |
| :--- | :--- | :--- |
| `id` | INT (PK) | 帖子唯一标识 |
| `u_id` | INT (FK) | 发布者 ID |
| `type` | ENUM | 类型 (lost/found) |
| `category` | VARCHAR | 物品分类 |
| `title` | VARCHAR | 物品名称 |
| `desc` | TEXT | 详细描述 |
| `img` | VARCHAR | 图片存储路径 |
| `contact_info`| VARCHAR | 该帖子的特定联系方式 |
| `status` | TINYINT | 状态 (0:展示中, 1:已结案) |
| `create_time` | DATETIME | 发布时间 |

## 4. 业务规则与逻辑
1.  **访问权限：** 游客可浏览首页和使用搜索功能；但点击“认领”、“联系”或“发布”时，必须引导至登录页面。
2.  **简单交互：** “我要认领”与“联系失主”不涉及在线聊天，仅作为联系方式的展示媒介。
3.  **数据过滤：** 搜索功能需实时响应或点击搜索按钮后局部刷新数据列表。
