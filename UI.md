## 一、 详细 UI 设计文档

基于 `image_58c67e.png` 的布局，我们对视觉细节、交互动效和规范进行定义。

### 1.1 视觉风格 (Visual Style)
*   **色调：** 建议采用“校园蓝”或“清新绿”作为主色调（如：`#3B82F6`），搭配大量留白，营造干净、可信赖的氛围。
*   **圆角规范：** 所有的卡片（Card）和按钮（Button）统一使用 `8px` 或 `12px` 的圆角，增加亲和力。
*   **字体：** 优先使用系统默认无衬线字体（PingFang SC, Microsoft YaHei）。

### 1.2 页面结构细节
| 模块 | 细化设计要求 |
| :--- | :--- |
| **导航栏** | 固定在顶部（Sticky），背景半透明高斯模糊。包含 Logo、搜索框、发布按钮、头像。 |
| **物品卡片** | 鼠标悬停（Hover）时产生微小的位移或阴影加深效果。 |
| **状态标签** | 招领卡片右上角显示“待认领”或“已结案”标签，用不同颜色区分。 |
| **弹窗 (Modal)** | 点击“认领”时弹出，背景遮罩，突出显示手机号并提供“一键复制”按钮。 |

## 二、 技术框架与组件选型

为了让 Claude 开发更高效、代码更健壮，建议采用以下**全栈“黄金组合”**：

### 2.1 前端：React + Tailwind CSS + Ant Design
*   **框架：** **React (Vite)**。起步快，组件化程度高，方便 Claude 生成模块化代码。
*   **样式：** **Tailwind CSS**。直接在 HTML 里写类名（如 `flex`, `rounded-lg`, `shadow-md`），Claude 对这种样式极其擅长。
*   **组件库：** **Ant Design (Antd)**。
    *   使用 `Modal` 组件实现联系方式弹窗。
    *   使用 `Form` 组件处理发布表单。
    *   使用 `Upload` 组件处理图片上传。

### 2.2 后端：Node.js (Express) + Sequelize
*   **框架：** **Express**。轻量简洁，是 Node.js 社区最成熟的框架。
*   **ORM (数据库工具)：** **Sequelize**。
    *   **理由：** 它可以让你用 JavaScript 对象的方式操作 MySQL，不用写复杂的原生 SQL。
    *   它能自动同步你定义的模型到数据库，非常适合快速迭代。

### 2.3 状态管理与认证
*   **认证：** **JWT (JSON Web Token)**。适合前后端分离的项目，安全且不需要在服务器存 Session。
*   **图片存储：** 建议 Claude 先实现**本地存储**（存储在 `public/uploads` 目录下），后续可轻松迁移到阿里云 OSS。
